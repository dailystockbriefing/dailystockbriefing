# 펩트론 데일리 브리핑 — 셋업 가이드

오픈채팅방에 **링크 하나만 고정**해두면, 멤버가 들어와 매 영업일 갱신되는
펩트론(087010)의 투자자 매매동향·공매도 잔고·매매 특이점을 봅니다. 카카오 API·봇 없이 동작합니다.

```
[GitHub Actions cron] --18:30 KST--> [collector.py / pykrx]
        --> data.json 커밋 --> [GitHub Pages 가 index.html 서빙] --> 멤버가 링크로 열람
```

## 파일 구성
| 파일 | 역할 |
|---|---|
| `index.html` | 멤버가 보는 단일종목 대시보드 |
| `collector.py` | KRX에서 종목 데이터 수집 + 특이점 신호 계산 (`collect_all()`) |
| `poll.py` | 게시 시점을 폴링하다 신규 데이터 감지 시 1회 수집 |
| `.github/workflows/update.yml` | 매 영업일 18:30 자동 실행·커밋 |
| `data.json` | 수집기 산출물 |

## 표시 항목
- **종가/등락** 헤더
- **오늘의 특이점** — 거래량 급증(20일 평균 대비 배수), 갭, 장중 변동폭, 종가 위치(마감 강도),
  20일 신고가/신저가, 외국인·기관 동반 순매수/매도, 외국인 연속 순매수, 공매도 비중 경고
- **투자자 매매동향** — 최근 10영업일 표(개인/외국인/기관 + 우측 스크롤로 금융투자·투신·사모·연기금·보험·은행·기타금융). **금액(억)/수량(주) 토글** 지원 — 수집기가 거래대금·거래량 두 벌을 모두 수집
- **공매도 잔고** — 최근 잔고수량과 **전 영업일 대비 수량 변화**(증감·%) + 최근 10영업일 잔고수량·비중 표 (T+2 기준, 금액은 미표시)

## 배포 (무료, GitHub Pages)
1. 저장소에 파일 업로드 (`update.yml` 은 `.github/workflows/` 경로).
2. 첫 데이터 생성:
   ```bash
   pip install pykrx pandas
   python collector.py
   git add data.json && git commit -m "init data" && git push
   ```
3. `Settings → Pages → Deploy from a branch → main / (root)` 저장.
4. `Actions` 탭에서 수동 실행으로 확인. 이후 평일 18:30(KST) 자동.
5. 생성된 `https://<아이디>.github.io/<저장소>/` 링크를 오픈채팅 공지로 고정.

## 종목 바꾸기 / 추가
`collector.py` 상단의 `TICKER / NAME / MARKET` 만 바꾸면 다른 종목으로 동작합니다.
여러 종목을 보려면 종목별 `data.json`(예: `087010.json`)을 만들고 페이지를 복제하거나,
한 페이지에서 종목 선택 탭을 추가하는 방식으로 확장하면 됩니다. 원하면 그 버전도 만들어 드립니다.

## 갱신 주기 (하루 2회 · 폴링)
매 영업일 두 번, **창이 열리기 직전에 잡 1개를 띄우고 잡 안에서 폴링**합니다.
(GitHub cron 은 최소 5분 단위 + 수 분 지연이 있어 "30초마다 발사"가 불가능하므로, 발사는 1회만 하고 폴링은 잡 내부 루프로 처리합니다.)

| 창 | cron(UTC) | KST | 폴링 | 감지 조건 |
|---|---|---|---|---|
| 투자자 동향 | `30 6 * * 1-5` | ~15:30 시작 | 30초 간격, 최대 9분 | 당일 투자자 데이터 게시 |
| 공매도 잔고 | `57 8 * * 1-5` | ~17:57 시작 | 60초 간격, 최대 16분 | 잔고 최신 기준일 진전 |

폴링은 매 주기 **가벼운 단일 프로브 1콜**로 게시 여부만 확인하고, 감지되는 순간에만 전체 수집을 1회 돌립니다.
창이 만료되면 마지막으로 한 번 수집하고, 커밋 단계가 **변경분이 있을 때만** 커밋하므로 빈 커밋은 생기지 않습니다.

## 비용 — 결론: 무료
- **공개 저장소면 GitHub Actions 는 무료·무제한.** 2026년 도입된 분당 $0.002 플랫폼 요금도 공개 저장소엔 적용되지 않습니다. 이 대시보드는 Pages 공개 링크로 공유하니 자연히 public → **폴링을 아무리 돌려도 0원.**
- **비공개로 둬도 Free 플랜 월 2,000분 안쪽:**
  - 잡은 *잠자는 시간 포함* 벽시계 시간으로 과금. 최악의 경우(신규 미감지로 창 끝까지 폴링) 투자자 9분 + 공매도 16분 = 하루 25분.
  - 25분 × 약 21영업일 ≈ **월 525분** → 2,000분의 1/4 수준. 실제로는 감지되면 즉시 종료(보통 1~3분)라 월 200~300분 정도.
- **KRX 호출량:** 프로브는 주기당 1콜(투자자 ≤18콜 + 공매도 ≤16콜) + 감지 시 전체 수집(약 12콜) → 하루 약 50~60콜. KRX 정보데이터시스템은 무료이며, 30~60초 간격이라 차단 우려도 낮습니다.

## 새로고침 버튼은 재수집하지 않습니다
페이지의 "최신 데이터 다시 받기"는 `location.reload()` 로 **이미 서버에 올라온 `data.json` 만 다시 내려받습니다.** `collector.py`/`poll.py` 를 돌리거나 KRX 를 호출하지 않습니다.
- 멤버가 몇 명이 누르든 정적 파일 재다운로드라 KRX 부하·Actions 사용량 0.
- 용도는 "저녁 갱신 후 최신 숫자를 기다리지 않고 바로 받기"이며, 장중 실시간 데이터를 끌어오는 게 아닙니다.

## 알아둘 점
- **공매도 잔고 T+2**: 18시에 나오는 잔고는 보통 2영업일 전 기준입니다(코드가 최근 데이터일로 자동 정렬).
- **GitHub 러너 해외 IP**: KRX 응답이 간헐적으로 막히면 국내 VPS/Self-hosted runner 에서
  `collector.py` 만 돌리고 결과를 푸시하는 구성을 권장합니다.

## 2단계 — 진짜 장중 패턴 (선택)
현재 특이점은 전부 **일별(EOD)** 데이터 기반입니다. 시간대별 거래량 집중, VWAP 이탈,
장마감 동시호가 물량, 체결강도 같은 **장중 패턴**을 보려면 증권사 API가 필요합니다.
- **한국투자증권 KIS Developers** (무료, 계좌 연동): REST `inquire-time-itemchartprice`(분봉),
  당일 체결, 호가 / WebSocket 실시간. 분봉을 마감 후 1회 받아 시간대별 분석을 사후 재구성하는 방식이 현실적.
- 키움 REST, LS증권 등도 유사 기능 제공.
필요하면 KIS 분봉 연동 + 장중 지표 패널을 얹은 버전으로 확장해 드립니다.

## 면책
KRX 데이터를 가공한 참고용 정보이며 정확성·완전성을 보장하지 않습니다. 투자 책임은 본인에게 있습니다.
